## API Comparison

### Kimi (Moonshot AI)

| Aspect | Details |
|--------|---------|
| **Service** | Claude AI for coding tasks |
| **Authentication** | OAuth 2.0 (JWT tokens) |
| **Token Location** | `~/.kimi/credentials/kimi-code.json` |
| **Token Type** | access_token + refresh_token |
| **Expiration** | April 2026 (future) |
| **API Endpoint** | `https://api.kimi.com/coding/v1` |

**Capabilities:**
- ✅ Chat/Conversation
- ✅ Code analysis & review
- ✅ Task completion
- ✅ Function calling
- ✅ Team collaboration (via custom scripts)
- ✅ Thinking/reasoning (streaming)

**Wrapper:** `kimi-wrapper.py` → 142 lines

**Usage:**
```bash
python3 kimi-wrapper.py chat "Your prompt"
python3 kimi-wrapper.py code "Code to analyze"
```

**Strengths:**
- Built for code-first workflows
- Deep thinking capabilities
- Real-time streaming
- Excellent for technical discussions

**Limitations:**
- Focused on coding/tech (though capable for general tasks)
- Requires OAuth setup (no simple API key)
- Rate limits not documented

### NVIDIA API

| Aspect | Details |
|--------|---------|
| **Service** | NVIDIA Inference Engine (NIM) |
| **Authentication** | Bearer token (API key) |
| **Token Location** | `~/.nvidia/credentials/nvidia-api.json` |
| **Token Type** | `nvapi-*` format |
| **Expiration** | Not specified (managed by NVIDIA) |
| **API Endpoint** | `https://integrate.api.nvidia.com/v1` |

**Models Available:**
1. `meta-llama/llama-2-70b-chat` — Llama 2 inference
2. `mistralai/mistral-7b-instruct` — Mistral 7B
3. `nvidia/nv-embed-qa-e5-large-v12` — Text embeddings (768-dim)
4. `nvidia/SegFormer` — Image segmentation

**Capabilities:**
- ✅ Chat/Inference (Llama, Mistral, etc.)
- ✅ Embeddings (semantic, RAG-ready)
- ✅ Vision models (SegFormer, object detection, classification)
- ✅ Function calling (via structured outputs)
- ✅ Multi-turn conversations
- ⏳ Streaming (API supported, wrapper ready)

**Wrapper:** `nvidia-wrapper.py` → 189 lines

**Usage:**
```bash
python3 nvidia-wrapper.py chat "Prompt" [--model llama]
python3 nvidia-wrapper.py embed "Text to embed"
python3 nvidia-wrapper.py models  # List all
```

**Strengths:**
- Multiple LLM options (Llama, Mistral, custom)
- Excellent for embeddings & semantic search
- Vision capabilities (unique vs Kimi)
- Simpler authentication (static API key)
- Mature inference engine

**Limitations:**
- No built-in code collaboration (but can be added)
- Vision models require specific payloads
- Embedding models need dimension awareness (768-dim)

```
3. Image processing

   for doc in docs/*.txt; do
     nvidia_embed "$(cat $doc)"
   done
```

## Security Audit

### Token Storage
✅ **Secure Locations:**
- Kimi: `~/.kimi/credentials/` (outside repo)
- NVIDIA: `~/.nvidia/credentials/` (outside repo)
- Both protected by file permissions (644)

✅ **Not in Git:**
- `.gitignore` should include `~/.kimi/` and `~/.nvidia/`
- Tokens never logged to stdout
- API keys hidden in wrapper output

### API Communication
✅ **HTTPS Only:**
- `https://api.kimi.com/coding/v1` (TLS)
- `https://integrate.api.nvidia.com/v1` (TLS)

✅ **Authentication:**
- Kimi: Bearer + JWT (expiring)
- NVIDIA: Bearer + static key

✅ **Secrets Management:**
- Tokens not in code
- Loaded from files at runtime
- Environment variables available (KIMI_ACCESS_TOKEN, etc.)

### Vulnerability Assessment
| Issue | Status | Notes |
|-------|--------|-------|
| Token hardcoding | ✅ CLEAR | Stored securely |
| Log exposure | ✅ CLEAR | UTF-8 safe, no token logging |
| Man-in-the-middle | ✅ CLEAR | HTTPS enforced |
| Token expiration | ✅ HANDLED | Kimi auto-refreshes, NVIDIA static |
| Credential access | ✅ PROTECTED | File permissions 644 |

## Resource Utilization

### Kimi API
- **Request size:** ~500 bytes (typical prompt)
- **Response time:** 0.5-3s (depending on thinking)
- **Concurrent requests:** Likely 10+
- **Rate limits:** Not specified by token

### NVIDIA API
- **Request size:** ~300 bytes (chat), ~100 bytes (embed)
- **Response time:** 0.1-2s (LLM), <0.5s (embeddings)
- **Concurrent requests:** Unlimited (per account)
- **Rate limits:** Likely ~100 req/min (standard)
- **Embedding cost:** ~12 tokens per average sentence
