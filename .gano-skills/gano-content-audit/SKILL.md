# Gano Content Audit — Skill de Auditoría de Contenido

Skill para detectar, auditar y reemplazar todo el contenido placeholder de gano.digital
con contenido real, profesional y orientado al mercado colombiano de hosting.

---

## Contexto de marca Gano Digital

**Propósito**: Proveedor de hosting WordPress de alto rendimiento y seguridad empresarial
para proyectos digitales serios en Colombia y LATAM.

**Diferenciadores clave:**
- Seguridad de nivel empresarial (Zero-Trust, WAF, cifrado post-cuántico en plan top)
- Soberanía digital — propiedad absoluta de tus datos
- Facturación en COP — sin conversiones de divisas
- Soporte en español 24/7
- IA integrada en cada plan (Gano Agent)

**Tono de voz:**
- Técnico pero accesible — habla con confianza, no con jerga inútil
- Directo — sin relleno, sin promesas vacías
- Colombiano — natural, cercano, sin falsedad corporativa

**Audiencias principales:**
1. PyMEs colombianas que necesitan presencia web profesional
2. Agencias digitales que gestionan múltiples clientes
3. Emprendedores con e-commerce o SaaS naciente
4. Empresas que han tenido problemas de seguridad o lentitud

## Estructura de páginas a crear

### /nosotros (About)
```
1. Historia breve — por qué existe Gano Digital
2. Misión: Soberanía digital para empresas colombianas
3. Equipo (si hay fotos reales disponibles)
4. Infraestructura: dónde están los servidores
5. Números reales: tiempo de operación, clientes actuales
6. Registros legales / NIT
7. CTA: Conoce nuestros planes
```

### /legal (Términos y Condiciones)
```
1. Términos de servicio hosting
2. Política de privacidad (Ley 1581 de 2012 — Colombia)
3. Política de uso aceptable (AUP)
4. SLA — Acuerdo de Nivel de Servicio
5. Política de reembolsos
6. Jurisdicción: Colombia, Cámara de Comercio de Bogotá
```

### /infraestructura
```
1. Especificaciones técnicas: NVMe Gen4, RAM dedicada
2. Ubicación de datacenters
3. Red: conexiones, redundancia, BGP
4. Seguridad física del datacenter
5. Certificaciones (ISO 27001 si aplica)
6. Diagrama de arquitectura
```

### /sla
```
1. Uptime garantizado: 99.9% mensual
2. Tiempo de respuesta de soporte por nivel (P1-P4)
3. Créditos por incumplimiento de SLA
4. Proceso de reclamación
5. Exclusiones (mantenimiento programado, fuerza mayor)
```

## Notas importantes

### Plugins de fase — NO ELIMINAR AÚN
Los plugins `gano-phase1-installer`, `gano-phase2-business`, `gano-phase3-content`,
`gano-phase6-catalog`, y `gano-phase7-activator` **despliegan el sitio web** con su
contenido y configuración. Deben permanecer instalados hasta que se confirme que cada
fase fue ejecutada exitosamente en WordPress.

**Solo eliminar DESPUÉS de:**
1. Activar cada plugin en WordPress (lo que ejecuta el installer)
2. Confirmar que el contenido/configuración de esa fase aparece en el sitio
3. Documentar qué instaló cada fase

### Quiz de Soberanía Digital
El quiz (`gano-sovereignty-quiz.js`) es un diferenciador de marca valioso. Mantener.
Conectar sus resultados a recomendación de plan específico.

### Chat IA
El chat (`gano-chat.js`) capta leads (nombre + WhatsApp). El texto del nudge automático
"detecté una vulnerabilidad en tu sitio" puede ser alarmista. Evaluar cambio a algo
más empático: "¿Cómo podemos ayudarte con tu proyecto digital?"

### Homepage Elementor y clases SOTA (Abril 2026)

- **Clases y mapeo** (repo): `memory/content/elementor-home-classes.md` — qué clase CSS va en cada widget; alinear con `gano-child` (`style.css` v1.0.1+).
- **Copy de referencia:** `memory/content/homepage-copy-2026-04.md`.
- **Menú:** el tema hijo registra `primary` además de `main` (Hello/Royal); en Elementor asignar el menú correcto al **Header** si el sitio no muestra navegación.
- **Git vs servidor:** cambios en Elementor viven en **BD**; el agente en GitHub solo puede preparar **código** (tema, MU-plugins, docs). Issues `[sync]` o solo servidor: dejar checklist en el issue o en `DEV-COORDINATION.md`.
- Orquestación masiva de issues: skill **`gano-github-copilot-orchestration`**.
