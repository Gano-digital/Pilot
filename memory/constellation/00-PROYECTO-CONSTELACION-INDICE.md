## 📊 Visión General Ejecutiva

Gano Digital es un **proveedor de hosting WordPress en Colombia** con modelo de negocio basado en integración con **GoDaddy Reseller Store**. El proyecto está estructurado en **5 fases de desarrollo** (2025-2027) impulsadas por un **ecosistema de 5 agentes autónomos** que colaboran en paralelo.

| Aspecto | Estado |
|--------|--------|
| **Fase actual** | 4 (Comercialización) |
| **Completion %** | 60% (Fases 1-3 ✅, Fase 4 en progreso, Fase 5 planeada) |
| **Agentes activos** | 5 (Claude, Cursor, VS Code, Antigravity, Obsidian) |
| **Revenue model** | GoDaddy Reseller (checkout marca blanca) |
| **Target market** | Empresas/pymes en Colombia requiriendo WordPress managed |
| **Competitive edge** | Soberanía digital + infraestructura SOTA + atención local |

4. Documentar decisión en la constelación

### Cuando Hay Bloqueador
1. Registrar en [[08-DEPENDENCIAS-RIESGOS#Bloqueadores Activos]]
2. Identificar agente propietario (o coordinador)
3. Crear subtarea de mitigation
4. Dar visibilidad a stakeholders via [[06-METRICAS-PROGRESO#Blockers & Risks]]

## 🎓 Glosario de Términos

| Término | Significado |
|---------|------------|
| **PFID** | Product Family ID (GoDaddy Reseller — mapeo producto WordPress → GoDaddy) |
| **RCC** | Reseller Control Center (backend de GoDaddy para administrar órdenes) |
| **Reseller Store** | Carrito marca blanca de GoDaddy (checkout al cliente final) |
| **Developer API** | API REST opcional (herramientas); no sustituye al Store; ver setup digital §1 |
| **Staging** | Ambiente de prueba (mirror de producción para validar cambios) |
| **OTE** | Official Test Environment (sandbox de GoDaddy antes de producción) |
| **MCP** | Model Context Protocol (protocolo de integración entre agentes) |
| **Fase X** | Etapa del roadmap de 5 fases (F1-F5) |
| **Dispatch Queue** | Sistema de coordinación de tareas entre agentes |
| **Artifact** | Salida de un agente (plan, código, test report, video) |
| **Blocker** | Tarea/decisión que bloquea el progreso de otra |
